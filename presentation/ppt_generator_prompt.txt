请生成一份关于软件项目“公司数据分析智能体（Company Data Analyst Agent）”的专业 PowerPoint 演示文稿。

受众：
- 教授 / 技术评审老师
- 希望快速理解项目的问题背景、系统架构、工作流程、数据来源和当前局限

语气：
- 技术性强，但表达清晰易懂
- 自信、扎实，不要过度营销化

页数：
- 8 页 PPT

设计风格：
- 浅色或白色背景
- 使用海军蓝 / 青绿色 / 石板灰作为强调色
- 整体风格偏现代数据产品展示
- 适当使用图标表示数据源、工作流、存储和部署
- 其中必须包含 1 页架构图、1 页路线图
- 避免空泛的创业式语言

每页内容结构如下：

1. 标题页
- 公司数据分析智能体
- 面向单一上市公司的深度研究工作流，用于生成有依据的分析师风格回答
- 副标题：Collect -> Explore / Analyze -> Hypothesize

2. 问题背景
- 公司研究通常分散在 SEC 文件、财务事实、市场数据和新闻稿中
- 单纯依赖 LLM 的回答可能跳过显式证据收集和分析过程
- 用户需要可追踪、可解释、带中间步骤的有依据输出

3. 方案概览
- Collector Agent 负责检索证据
- EDA Agent 负责分析证据
- Analyst Agent 负责综合生成最终回答
- Streamlit 前端清晰展示每个阶段

4. 数据来源
- SEC 文件：10-K、10-Q、8-K
- SEC CompanyFacts：结构化财务指标
- Yahoo Finance：市场数据
- NVIDIA 新闻稿：公司叙事与公告信息
- 强调定性证据与定量证据两条路径

5. 系统架构
- 前端：Streamlit
- 编排器：LangGraph
- 工具：检索、EDA、图表生成、最终回答构建
- 存储：SQLite + 本地产物
- 部署目标：Google Cloud Run
- 可视化流程：User -> UI -> Workflow -> Tools -> Storage/Data Sources -> Final Answer

6. 检索与分析
- 定性检索：基于财报和新闻稿切块
- 定量检索：基于财务指标和市场数据
- 确定性 EDA 工具：
  - financial trend
  - market reaction
  - text themes
  - charts
- 强调输出是 grounded 且可检查的

7. Demo / 输出效果
- 用户提出一个关于 NVIDIA 的问题
- Collect 页展示收集到的证据
- Explore 页展示分析发现和图表
- Hypothesis 页展示带来源依据的最终分析回答

8. 优势、局限与下一步
- 优势：流程透明、多智能体设计、双路径检索、已部署
- 局限：当前对 NVDA 支持最强，新闻稿覆盖偏公司定制，编排仍较启发式
- 下一步：更智能的路由、更强的定性解释能力、更广的公司覆盖

- 如果合适，请补充 speaker notes
- 第 4、5、6、8 页请给出建议图示
- 每页控制在 5 个 bullet 以内
