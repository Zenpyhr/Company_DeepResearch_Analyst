公司数据分析智能体
面向单一上市公司的深度研究工作流，用于生成有依据的分析师风格回答
Collect -> Explore / Analyze -> Hypothesize

为什么要做这个项目
- 公司研究信息分散在财报、财务事实、市场数据和公司公告中
- 用户需要带证据、带分析过程的回答
- 项目目标是做一个 AI 公司尽调助手，而不是交易机器人

方案概览
- Collector Agent 检索相关证据
- EDA Agent 使用工具分析证据
- Analyst Agent 生成最终有依据的回答
- Streamlit 前端展示每个阶段

数据来源
- SEC 文件：10-K、10-Q、8-K
- SEC CompanyFacts
- Yahoo Finance 市场数据
- NVIDIA 新闻稿
- 定性证据路径 + 定量证据路径

系统架构
- 前端：Streamlit
- 编排器：LangGraph
- 存储：SQLite + 本地产物
- 部署：Docker + Cloud Run
- 输出：回答、引用、图表、memo 产物

检索与 EDA
- 定性检索：基于财报和新闻稿切块
- 定量检索：基于财务指标和市场数据记录
- 分析工具：财务趋势、市场反应、主题提取、图表生成
- 底层以稳定确定性流程为主，并结合轻量 LLM 润色

用户演示流程
- 刷新 ticker 数据
- 输入一个分析师风格问题
- Collect 页展示证据
- Explore 页展示发现和图表
- Hypothesis 页展示最终 grounded memo

当前优势与下一步
- 优势：流程透明、输出 grounded、多智能体结构、已部署应用
- 局限：当前对 NVDA 支持最强，新闻稿覆盖有限，编排仍偏启发式
- 下一步：更智能的路由、更强的检索、更广的公司支持
