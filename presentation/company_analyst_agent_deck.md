# 公司数据分析智能体

## 版本 A：8 页课程汇报 / Demo PPT

### 第 1 页 - 标题
**公司数据分析智能体**  
面向单一上市公司的深度研究工作流，用于生成有依据的分析师风格回答

- 当前一次主要聚焦一家上市公司，现阶段对 `NVDA` 的支持最完整
- 系统整合了运行时数据采集、多智能体编排、检索、EDA 分析和最终综合回答
- 输出结果包括：基于证据的回答、关键发现，以及可视化图表产物

演讲备注：
这个项目更适合被描述为“AI 公司尽调助手”，而不是交易机器人。目标是结合定性和定量证据，帮助用户完成公司研究。

### 第 2 页 - 问题背景
**为什么要做这个项目**

- 公司研究信息分散在财报、监管文件、财务数据、市场数据和公司公告中
- 纯聊天式大模型回答往往会跳过显式证据收集和中间分析过程
- 用户需要一个能够清楚展示以下过程的系统：
  - 收集了什么数据
  - 做了什么分析
  - 为什么最终结论是有依据的

演讲备注：
本项目强调“分析流程可见”。系统不会直接从问题跳到最终文字答案，而是把收集、EDA 和综合推理拆成多个显式阶段。

### 第 3 页 - 方案概览
**核心思想：Collect -> Explore / Analyze -> Hypothesize**

- `Collector Agent` 根据用户问题检索并收集证据
- `EDA Agent` 对收集到的证据执行确定性分析工具
- `Analyst Agent` 将分析结果综合成最终的、有依据的回答
- Streamlit 前端将每个阶段分标签页展示，增强透明度

建议图示：
用户问题 -> Collector -> 证据包 -> EDA -> 分析发现 -> Analyst -> 最终回答

### 第 4 页 - 数据来源
**运行时数据管线**

- SEC 文件：`10-K`、`10-Q`、`8-K`
- SEC CompanyFacts：结构化财务指标
- Yahoo Finance：市场价格与成交量历史数据
- NVIDIA 新闻稿：公司叙事与公告信息

数据使用方式：
- 定性证据：财报 / 文件切块 + 新闻稿切块
- 定量证据：财务指标记录 + 市场数据记录

演讲备注：
项目的一个亮点是同时设计了定性和定量两条证据路径，因此既能回答风险和叙事问题，也能回答财务趋势问题。

### 第 5 页 - 系统架构
**系统架构设计**

- 前端：Streamlit 应用
- 编排层：LangGraph 工作流
- 工具层：检索、EDA 分析、图表生成、最终回答构建
- 存储层：
  - SQLite 用于结构化记录和文本切块
  - 本地产物用于保存图表、备忘录和中间结果
- 部署目标：Google Cloud Run

建议图示：

- 工作流 / 编排器
- 工具层
- 存储与产物
- 外部数据源

### 第 6 页 - 检索与 EDA 设计
**双路径检索 + 确定性分析**

- 定性检索：
  - 在 SEC 文件切块和新闻稿切块中进行搜索
  - 主要用于风险、管理层叙事、战略、增长主题分析
- 定量检索：
  - 获取财务指标和市场数据记录
  - 主要用于趋势分析和市场反应分析
- EDA 工具：
  - `financial_trend_tool`
  - `market_reaction_tool`
  - `text_theme_tool`
  - `chart_tool`

演讲备注：
一个关键设计选择是：底层检索和分析尽量保持确定性，以确保系统稳定、可解释；LLM 更多用于润色、总结和最终综合表达。

### 第 7 页 - Demo 流程
**用户实际看到什么**

- 用户先选择或刷新某个股票代码的数据
- 用户提出一个问题，例如：
  - “NVIDIA 最近的财务表现是否支持 AI 增长叙事？目前仍然突出的风险是什么？”
- 系统返回：
  - Collect 页：研究目标、检索说明、选中的工具、数据来源数量
  - Explore / Analyze 页：分析发现、指标、支持记录、图表
  - Hypothesis 页：带来源依据的分析师风格总结

### 第 8 页 - 优势、局限与后续工作
**当前价值与未来路线**

优势：
- 多阶段流程清晰
- 同时结合文本证据和财务数据
- 输出有依据，且中间分析过程可见
- 已有可部署应用和较完整代码结构

当前局限：
- 当前对 `NVDA` 的支持最强
- 新闻稿覆盖仍然偏公司定制化
- 核心编排逻辑仍以启发式和确定性规则为主
- 更强的向量检索和更智能的 LLM 规划仍属于后续工作

下一步：
- 提升混合问题的路由能力
- 增强定性内容理解能力
- 加强重试逻辑和检索规划
- 扩展到更多公司

## 1 分钟项目总结

本项目是一个公司尽调智能体，能够基于定性和定量证据回答分析师风格的问题。系统在运行时从 SEC 文件、SEC CompanyFacts、Yahoo Finance 和 NVIDIA 新闻稿中收集数据，然后执行三阶段流程：Collector、EDA 和 Analyst。系统设计的重点是透明性，因此前端不仅展示最终答案，也展示收集了哪些证据、做了哪些分析，以及最终结论是如何形成的。当前版本对 NVIDIA 的支持最完整，底层以稳定的确定性流程为主，并结合轻量的 LLM 做润色和综合表达。
